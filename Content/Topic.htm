Merging - Two User/Computer Test Cases

    

Both users check-out the same file and make non-conflicting changes. Both users check in changes. Auto resolve these changes and make sure both copies are updated correctly.

            

Both users check-out the same file and make conflicting changes. Verify in the merge tool that the correct server version and local version is shown. Verify that the changes you make are applied to both copies.

            

Both users check-out the same file and make conflicting changes. Have one user choose to undo local changes. Verify the local copy is updated correctly.

            

Both users check-out the same file and make conflicting changes. Have one user choose to discard external changes. Verify the server copy is updated correctly.

            

Both users check-out the same file. One user deletes the file , the other user checks in the file before doing a Get Latest.

            

Both users check-out the same file. One user renames the file , then the second user checks in the file before doing a Get Latest.

            

Both users have files checked out and one user renames the folder holding the files - and checks in changes.  Second user checks in changes before doing a Get Latest.
